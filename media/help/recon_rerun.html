Re-running a reconciliation task

You have already run a task of this type for this dataset.

During the Review step after running a reconciliation task, any decisions made ('no match' or 'closeMatch') are recorded in the WHG database. All hits presented for each given record are marked 'reviewed' and in the case of a 'closeMatch' decision, new place-link and possibly place-geom records are created.

If review of a prior task is incomplete and you run the same task type again (e.g. wikidata, tgn), by default we will skip submitting any place records already reviewed. For example: in a previous task, 200 records got 1 or more hits, and you made decisions on 50 of the records. If you submit the same type of task, the 50 records already reviewed will be skipped, and decision results already made for them will be retained.

If you never made any review decisions for the earlier task(s), ALL records will once again be submitted. You should delete the previous task to avoid confusion.

If review of results from the previous task(s) is complete, the option to run it again is disabled to prevent duplication of effort.

If you really want to submit all your records again (and review hits for all), you should delete any previous tasks of that type on the Reconciliation tab. That choice of task will then be enabled here again.
